# CodeT5 Fine-tuning Complete Implementation Guide
## Executive Summary

This is a **complete, production-ready** implementation for fine-tuning CodeT5 on the MegaVul dataset for vulnerability detection.

---

## 📦 Deliverables

### Core Scripts (in `scripts/` directory)

| Script | Purpose | Input | Output |
|--------|---------|-------|--------|
| `01_preprocess.py` | Load MegaVul JSON, clean data, create train/val/test splits | `Data/raw/megavul_simple.json` | Processed JSONL files |
| `02_train.py` | Fine-tune CodeT5 model on vulnerability detection | Processed data | Trained model weights |
| `03_evaluate.py` | Evaluate model on test set, generate metrics & plots | Test data | Metrics, confusion matrix, ROC curve |
| `04_inference.py` | Run inference on new code (interactive/batch/file mode) | Code snippets | Predictions & confidence scores |
| `05_model_utils.py` | Model analysis, benchmarking, quantization | Trained model | Performance stats, optimized versions |
| `06_unified_analyzer.py` | Integrated analysis combining all models + CWE + localization | Code | Detailed vulnerability report |

### Configuration Files

| File | Purpose |
|------|---------|
| `requirements.txt` | All Python dependencies with versions |
| `setup_environment.bat` | Automated Conda environment creation (Windows) |
| `accelerate_config.yaml` | Multi-GPU training configuration |
| `README.md` | Comprehensive documentation with step-by-step guide |

## 🚀 EXECUTION STEPS

### Step 1: Environment Setup (5 minutes)

**Windows:**
```batch
setup_environment.bat
conda activate codet5_vul
```

**Linux/Mac:**
```bash
conda create -n codet5_vul python=3.11 -y
conda activate codet5_vul
pip install -r requirements.txt
conda install pytorch pytorch-cuda=12.1 -c pytorch -c nvidia -y
```

### Step 2: Preprocessing (10-20 minutes)

```bash
python scripts/01_preprocess.py \
    --raw_data Data/raw/megavul_simple.json \
    --output_dir Data/processed \
    --train_split 0.7 \
    --max_code_length 1024
```

**Outputs:**
- `Data/processed/train.json` (~7,000-10,000 samples)
- `Data/processed/val.json` (~1,500-2,000 samples)
- `Data/processed/test.json` (~1,500-2,000 samples)
- `Data/processed/dataset_info.txt` (Statistics)

### Step 3: Training (2-6 hours depending on GPU)

**Single GPU (RTX 4090: ~2-4 hours):**
```bash
python scripts/02_train.py \
    --train_data Data/processed/train.json \
    --val_data Data/processed/val.json \
    --output_dir models/codet5_base_vuln_detector \
    --batch_size 16 \
    --num_epochs 5 \
    --learning_rate 2e-5
```

**Multi-GPU (2+ GPUs):**
```bash
accelerate launch scripts/02_train.py \
    --train_data Data/processed/train.json \
    --val_data Data/processed/val.json \
    --batch_size 16
```

**If OOM Error:**
```bash
python scripts/02_train.py \
    --batch_size 8 \
    --gradient_accumulation_steps 2 \
    --max_length 384
```

**Monitor Progress:**
```bash
tensorboard --logdir models/codet5_base_vuln_detector/logs
# Visit http://localhost:6006
```

### Step 4: Evaluation (5-10 minutes)

```bash
python scripts/03_evaluate.py \
    --model_dir models/codet5_base_vuln_detector \
    --test_data Data/processed/test.json \
    --output_dir outputs/evaluation
```

**Outputs:**
- `outputs/evaluation/metrics.json` - Full metrics
- `outputs/evaluation/confusion_matrix.png` - Confusion matrix visualization
- `outputs/evaluation/roc_curve.png` - ROC curve
- `outputs/evaluation/predictions.json` - Per-sample predictions

**Expected Metrics:**
- Accuracy: 75-85%
- F1 Score: 70-80%
- Precision: 70-80%
- Recall: 70-85%

### Step 5: Test Inference (1 minute)

**Interactive Mode:**
```bash
python scripts/04_inference.py \
    --model_dir models/codet5_base_vuln_detector \
    --mode interactive
```

Then enter code when prompted and type 'END'.

**Quick Test:**
```bash
python scripts/04_inference.py \
    --model_dir models/codet5_base_vuln_detector \
    --mode batch \
    --code "def foo(): strcpy(a, b)"  # Should predict vulnerable
```

### Step 6: Model Analysis (2 minutes)

```bash
python scripts/05_model_utils.py \
    --model_dir models/codet5_base_vuln_detector \
    --action analyze
```

Outputs model parameters, size, memory requirements.

## 🔧 TROUBLESHOOTING

### CUDA Out of Memory
```bash
# Reduce batch size
python scripts/02_train.py --batch_size 8

# Reduce sequence length
python scripts/02_train.py --max_length 384

# Use gradient accumulation
python scripts/02_train.py --gradient_accumulation_steps 4
```

### Model Download Failed
```bash
# Check internet connection
ping huggingface.co

# Manually download
huggingface-cli download Salesforce/codet5-base --local-dir ./models/codet5-base

# Use local model
python scripts/02_train.py --model_name ./models/codet5-base
```

### Data Preprocessing Fails
```bash
# Verify JSON is valid
python -c "import json; json.load(open('Data/raw/megavul_simple.json'))"

# Use smaller test dataset
head -c 100000000 Data/raw/megavul_simple.json > Data/raw/test_sample.json
python scripts/01_preprocess.py --raw_data Data/raw/test_sample.json
```
